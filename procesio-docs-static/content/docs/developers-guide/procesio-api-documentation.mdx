Our REST APIs allow you to integrate with PROCESIO from any other application. We expose 2 APIs: Proxy API and Web API. Use the Proxy API for connecting to the Authentication Service, and the [Web API](https://webapi.procesio.app/swagger/index.html) to perform operations on your application, such as running a process, or getting a data type configuration.

# Proxy API

Use the [Proxy API](https://auth.procesio.app/swagger/index.html) to reach the Authentication Service. The Authentication Service exposes methods such as authenticate, refreshToken, password change, and password update.

In order to be able to perform any operation on the PROCESIO platform through the Web API, you need to first get an access token from the Proxy API. In order to obtain an access token you can use the POST /api/Authentication method.

The **POST** /api/Authentication method requires the following parameters:

- **realm**: your tenant name (all users are added to "procesio01")
- **grant\_type**: password
- **username**: your user name
- **password**: your user’s password
- **client\_id**: procesio-ui
- **client\_secret**: (can be left empty)

If the request is successful, you will receive an access token in the response which you will be able to use to access the Web API services.

# Web API

Use the [Web API](https://webapi.procesio.app/swagger/index.html) to interact with the PROCESIO platform. To be able to send requests to the Web API, you need to be authorized with the access token received from the Proxy API (see the previous chapter regarding the Proxy API). The **Authorize** button can be found in the upper right corner.

### How to run a process from an external application?

### Prerequisites

To run the process using a API Key we will require:

- Generate a API Key in PROCESIO.

![](/images/docs/oODNXNcdEbsUsKjsWqCvw_image.png)

- PROCESIO API endpoints can be found here. Additionally we also provide [Swagger for our Web API](https://webapi.procesio.app/swagger/index.html) endpoints which can be tested here.
- [Postman](https://www.postman.com/downloads/) for sending the request.

### Postman

Configure the request:

&#x20; 1\.    Select Post Method mention the base URL and the endpoint

- **Base URL**: https\://webApi.procesio.app
- **Endpoint**: /api/Projects/e6bb2d64-2f96-4e43-a84b-b25a08c0764d/run (includes the Process ID)

&#x20; 2\.    In Postman from Headers include:

| **Headers** | **Value**     |
| ----------- | ------------- |
| key         | API key name  |
| value       | API key value |
| workspaceId | workspace Id  |

![](/images/docs/HJNCC4vbXk8qodE_CA1Ah_image.png)

 `

 `
