Use this action to execute a SQL command in Microsoft SQL.&#x20;

![](/images/docs/4Cc8mVlmk7uvW66kl7YB-_image.png)

First of all, you'll have to enter credentials for the **Execute Command** action. You can find out how to do that [here](https://docs.procesio.com/sql-credential).&#x20;



![](/images/docs/HHT894hjOlpHEoKjQ_T_0_screenshot-2023-02-22-at-171621.png)

**How to configure the Execute Command action?**

Go to the Process Designers section and open a new or an existing process. Actions configuration can be performed once they are dragged and dropped on canvas.

**Step 1**: Drag the **Execute Command** action from Toolbar and drop it on canvas; click on the action to open the **Properties Panel**.

**Step 2**: You can edit the Action Name.

**Step 3:&#x20;**&#x53;elect a credential for the database server [(see here how to create one)](https://docs.procesio.com/sql-credential)

**Step 4:&#x20;**&#x43;lick on **Execute Command (not configured)** card to go to the configurations steps.

**Step 5: &#x20;**&#x54;ype your SQL query in the editor.

> Note: you can always add process variables to your query. Press INSERT/Ctrl+I (Windows) or Cmd+I (Mac).

**Step 6:&#x20;**&#x41;dd the output variable. Note: it has to be defined as a single-value object/integer.&#x20;

`In the output variable, you will see the number of rows affected by the executed command.`

![](/images/docs/J0nPVi73Bv-jMBsxZHW35_screenshot-2023-02-22-at-172822.png)

![](/images/docs/YZ-soEnDhYGKXbmE16UEr_screenshot-2023-02-22-at-172902.png)

**Step 7:**   **Save**, **Validate** and **Run** the process.&#x20;

**Step 8:** Click **Check Instance&#x20;**&#x74;o view the results.

## Note

## Using SQL Parameters

When writing SQL commands in **Execute Command**, you can use **SQL parameters** to safely send values from your process to the database.

Instead of inserting variables directly in the SQL command, define parameters and map them to values in the configuration panel (as shown in the screenshots above).

For example, instead of writing:

> INSERT INTO Clients (FirstName, Email)VALUES ('\&#123;\&#123;client.firstName&#125;&#125;', '\&#123;\&#123;client.email&#125;&#125;')

You can use parameters:

> INSERT INTO Clients (FirstName, Email)VALUES (@firstName, @email)

Then map the parameters in the action configuration:

| Parameter | SQL Type | Value                  |
| --------- | -------- | ---------------------- |
| firstName | NVarChar | \&#123;\&#123;client.firstName&#125;&#125; |
| email     | NVarChar | \&#123;\&#123;client.email&#125;&#125;     |

Using parameters helps you:

- avoid SQL syntax errors caused by special characters like '&#x20;
- reduce the risk of SQL injection&#x20;
- simplify SQL commands that use process variables&#x20;
- make workflows easier to maintain&#x20;

If a parameter value is **null**, PROCESIO automatically sends a **SQL NULL** value to the database.
