Procesio allows you to work with several types of files, the most useful of which being Comma separated files, that use the .csv extension.

### Scenario

The user wishes to use the process to export data in the form a .csv file.

### How to[﻿](https://docs.procesio.com/duplicate-identification#lm-how-to)﻿[﻿](https://docs.procesio.com/list-filter-simple#oy-how-to)﻿

&#x20;   Step 1.    [Create a new process](https://docs.procesio.com/creating-a-basic-process) and give it a name.

![](/images/docs/I6xBqTdvXVRL__lg_nQP-_image.png)

&#x20;    Step 2.    [Add the needed variables](https://docs.procesio.com/working-with-process-variables).

For the process used in our example we will require the following variables:

- &lt;%data%&gt; - the input list\&lt;Object> variable that will store the JSon.
- &lt;%export%&gt; - the output variable of type File.

![](/images/docs/2s6KtVJczKHSisx2z9b8A_image.png)

&#x20;    Step 3.   Click the **Export to CSV** action and add the following configuration.

- Object to export\* - will hold the \&lt;%data%> variable.
- Column delimiter - will ask for the type of delimiter used in the newly created file.
- Export header - the checkbox gives us the option to also export the headers.
- File name\* - will be the name of the exported file, the extension will be added by Procesio.
- Output file\* - will contain the fil variable we created  in Step 2.&#x20;

<Callout type="info">
The **File name\*** should be the same as the **Output file\*** name. Check the screenshot below.
</Callout>

![](/images/docs/WDcBiAbdakXEpnWaEHNOW_image.png)

&#x20;   Step 4.   **Save**, **Validate&#x20;**&#x61;nd **Run&#x20;**&#x74;he process.

&#x20;   Step 5.   You will be prompted to add the data for our input variable. The input will be of type JSon. Click **Run**.
