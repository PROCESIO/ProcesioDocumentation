At PROCESIO, the public roadmap is crucial for transparency, community engagement, and aligning our development priorities with the needs of our users.

![](/images/docs/jigsCwce7m4mmB6sGjeyO_public-roadmap2.png)

**Import/Export:**

Ability to import and export processes and resources between workspaces.

**Process Library:**

Ready-to-use process templates and related resources available for download in users' workspaces. This component is intended to become a marketplace for users to share their work with the PROCESIO community, with a rewarding system in place as a final milestone.

**Global Variables:**

Variables that retain their value over time and can be updated via processes or manually through the user interface.

**Test Action:**

Users can test standalone actions within PROCESIO by simply clicking the test action button.

**OAuth:**

Users can grant limited access to third-party applications or services without giving away their login credentials, enhancing the user experience and platform usability.

**ErrorPort:**

In the event of action failure or error, users will have the ability to continue the process on a dedicated error branch, making it easier to handle errors within the process

**Lower Processing Time per Action:**

PROCESIO is undergoing a refactorization that will improve overall action processing speed.

**Horizontal Scaling:**

PROCESIO will be able to scale up and down according to users' resource requirements.

**Message Queue Recovery:**

If a service breaks down, running processes will no longer be affected.
