**PROCESIO** allows you to connect to other platforms in a No-Code way by using configurable credentials.&#x20;

These credentials can be used with various actions such as **Call API&#x20;**&#x6F;r **Send Email** to connect to **REST APIs**, **SMTP servers**, **FTP servers**, and even **SQL Server&#x20;**&#x64;atabases.

<Callout type="info">
For example, if you want to connect to the **GitHub** **API**, you should configure the authentication only once in the **Credentials Manager**. Afterwards, simply select the credentials configuration in every **Call API** action that you plan to use while working with the API.
</Callout>

## Creating a credential

We will use [Dictionary API](https://dictionaryapi.dev/) to search for the meaning of an English word in an online dictionary.

**To create the credential:**

- Click the **Credential Manager** section from the main menu.

![](/images/docs/ZgLh-3imP9HENu-Frorr0_tdy.png)

- Hit the **Create new** button to start configuring.

![](/images/docs/lncj44FecloR5rwrkwRGR_tdy.png)

- Fill in a name, description and choose the credential type from the drop-down.&#x20;

<Callout type="info">
Even though the **description** field is optional, we suggest that you provide a description for each and every credential you create, just to make it easier for others to understand how something works or the purpose of a credential.&#x20;
</Callout>

- Then click **Next step.**

![](/images/docs/ZiFGwuz0YWMb70l_7-l_f_tdy.png)

- For now, let's add the **URL** of the Dictionary API that we found in its [documentation](https://dictionaryapi.dev/) and select **No authentication** for the authentication method since there is none.

![](/images/docs/UJVjx8vunl4Ai8iAPT3Dr_tdy.png)

- Clic&#x6B;**&#x20;Save** and there you have it.

## Testing a credential

If you check the **Credential Manager**, you will see that your new credential just showed up.

As you can see, the credential is <span style={{color:'#eb144c'}}>not validated</span> (we skipped that part on purpose when creating it 😈)

![](/images/docs/2QECbiEF7s4Z63GXeT_oQ_tdy.png)

To validate it you have two options:

- Validate the credential by pressing **Test connection** when creating it. (*like we should have done*)
- Validate it after creation. (*also good*)

Since the credential is already here, we'll go for the second option.&#x20;

**Let's validate it now:**

- Click **Edit** to review the credential configuration

![](/images/docs/TZwMcL4Kdji1p81Iwe-kV_tdy.png)

- Click **Next step** to reach the validation view

![](/images/docs/9hmY2vgoil85wDy2_t8Xw_tdy.png)

- Fill in the **Test endpoint** and choose the required **HTTP method**

We want **GET** the meaning of "automation" so we will use `/automation` as the **Test endpoint**.

![](/images/docs/jXmQAX2GzKAhTrQhNlmpa_tdy.png)

- Hit **Test connection**, check the output, and then **Save** if everything went OK

![](/images/docs/KHsbrfS1ex-y24oXui50f_empty.png)

- After saving, you'll notice the credential has been updated

![](/images/docs/xjVHFCuqoM-AqTqIj6DA7_tdy.png)

As you can see, the credential is validated so we're all set for now.
