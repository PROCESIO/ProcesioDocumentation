When you build a new process, the **Run** button is disabled unless the process is both **saved** and **valid**.&#x20;

**Running a process**

There are currently three ways to run your process:

- &#x20;From the Topbar, by pressing  the **Run** button.

![](/images/docs/Wwa1LMw9jxc5w2mRsQbLa_one.png)

- From the Validation confirmation message by pressing the **Run process** button.

![](/images/docs/5Ma1xH11bS-MWjk2LzSNz_three.png)

- From the process list, in **Process Designer**.

![](/images/docs/1dH8v4g0YEHVhp5pWLhfG_two.png)

**Process flow without Input process variables**

- If the process flow has no variable set as **Input**, the process flow will simply start running once you hit the **Run** button.

**Process flow with Input process variables**

- If the process flow has variables declared as **Input**, once you hit the **Run** button, a dialog will pop up requiring values for all **Input&#x20;**&#x76;ariables in order to initiate the process execution.

![](/images/docs/yl6PErCnX91HuJ1EuVAtT_tdy.png)

The values provided for the **Input** variables should be of the same type. This means that for **File** variables, you will have to upload files and for **Data Model** variables, you will be able to set values using JSON.

Once you fill in the values, hit the **Run&#x20;**&#x62;utton so that the dialog closes and the process flow starts running.&#x20;

### **Process instances&#xA0;**

When you hit the **Run** button, a notification will be shown that the process completed.

![](/images/docs/56O4x91YVaGhUMiFW_SFL_processcompleted.png)

- Click on **Check instances** to go to instances details. 

OR

- Click on the contextual menu from the Process Designer and go to **View Instances**:

![](/images/docs/g4nd6zLs7d5Fp_dTn3X6s_screenshot-2022-05-31-at-195232.png)

In the **Instances** page, you will see all the instances generated for each run of the process, with each having details like: 

- **Instance Id**: for every new run, a new instance of the process is created with a unique id;
- **Actions consumed**: the number of actions that were used for that run of the process;
- **Started on**: the date when the process was initiated by the user (when the instance was created);
- **Last updated**: when the last action within the process was processed; 
- **Submitted by**: username of the person who launched the process;
- **Status**: the current status of the process

![](/images/docs/-iXqAhiSmwmL_t45mBFMW_screenshot-2022-05-31-at-195609.png)

Click on an instance to expand and view **Input&#x20;**&#x61;nd **Output&#x20;**&#x76;ariables (if any were declared)

![](/images/docs/lTs6_waU36cVle59bet1u_screenshot-2022-05-31-at-195736.png)

Click on **Show details&#x20;**&#x74;o access the **Instance View** to see the instance and the state of each action.

In the **Instance View**, click on an **action&#x20;**&#x74;o view details about it: status, started on, when it was last updated, input and output process variables' values if there were any declared for the process.&#x20;

![](/images/docs/O1Mk5dIVkyjsHDVLRd0KM_runprocessdemo.png)
