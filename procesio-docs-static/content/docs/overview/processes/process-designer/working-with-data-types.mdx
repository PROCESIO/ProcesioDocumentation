Each process variable has a data type. **PROCESIO** supports the following data types:

- **Primitives:&#xA0;**<span style={{color:'#1c417e'}}>String</span>, <span style={{color:'#0039e3'}}>Integer</span>, <span style={{color:'#064907'}}>Float</span>**&#x20;/** <span style={{color:'#064907'}}>Double</span>, <span style={{color:'#7239e3'}}>Boolean</span>, <span style={{color:'#ff9307'}}>DateTime</span>, <span style={{color:'#007364'}}>Json</span>, GUID, **Object**.
- **Custom:&#xA0;**<span style={{color:'#007364'}}>Data Model</span> (custom data type with one or more attributes), <span style={{color:'#0671d6'}}>File</span>.&#x20;

### **The&#x20;**<span style={{color:'#0671d6'}}>File</span>**&#x20;data type**

**<span style={{color:'#0671d6'}}>File</span>&#x20;**&#x69;s a <span style={{color:'#007364'}}>Data Model</span> that contains the following attributes:

- **path**: path to file (set on upload)
- **name**: name of the file
- **mime-type**: file type
- **size**: file size
- **hash**: file checksum

Use a <span style={{color:'#0671d6'}}>File</span> variable type as process input to be able to create processes that use dynamic files as attachments in the **Send Email** action. Or use it as an output variable to map to it a file that you get from the **Call API** action. Or many others! 🚀

### **Mapping <span style={{color:'#ff9307'}}>variables</span> to other data types**

Be mindful of the data type compatibility when passing values from one process variable to the property of an action. Each action property expects a given data type. This means that the variable type must match the expected property type of the action. We refer to this as data mapping.&#x20;

Don't worry, there are tooltips that tell you what data type is expected in input:



![](/images/docs/jEkkGB2gheeo56I3fWhwI_screenshot-2022-05-31-at-201815.png)

&#x20;Also, each variable will have a color-coded tag to easily map the variable to its data type:

![](/images/docs/asOaSyvCrF4ooiLyIu3Wf_types.png)



The <span style={{color:'#ff9307'}}>variables</span> used at a process level are called process <span style={{color:'#ff9307'}}>variables</span>. Each process may have:

1. **Input** process <span style={{color:'#ff9307'}}>variables</span>
2. **Output** process <span style={{color:'#ff9307'}}>variables</span>
3. **Process Variables** used within the process

If the process needs certain dynamic data in order to start, you need to configure process <span style={{color:'#ff9307'}}>variables</span> that have the “Input” tag enabled. Declare <span style={{color:'#ff9307'}}>variables</span> as “Output” if there is certain data that you expect to get back from the process once the process has been executed successfully.

 You need to consider the following when mapping input/outputs to process <span style={{color:'#ff9307'}}>variables</span>:

- **PROCESIO** does not support mapping data from one type to another thus you need to map data of the same data types. You can always create a new process variable of the corresponding data type if needed.
  - **Exception**: conversion from Integerto <span style={{color:'#064907'}}>Float</span> is allowed, as well as conversion from any primitive data type to <span style={{color:'#1c417e'}}>String</span>. 
- Pay attention to the  **List\&lt;>** type of a process variable. Even though 2 process <span style={{color:'#ff9307'}}>variables</span> may have the same data type, you cannot map a process variable declared as a single value to a process variable declared as a list.

**Mapping Rules**

- <span style={{color:'#1c417e'}}>String</span>: you can map any type of variable and concatenations.  
  - **Exception**: A <span style={{color:'#007364'}}>Data Model</span> cannot be mapped to a <span style={{color:'#1c417e'}}>String</span> property. Mapping of a <span style={{color:'#007364'}}>Data Model</span> attribute to a <span style={{color:'#1c417e'}}>String</span> property is allowed.
- <span style={{color:'#0039e3'}}>Integer</span> property: you can map only  <span style={{color:'#0039e3'}}>Integer</span> <span style={{color:'#ff9307'}}>variables</span>
- <span style={{color:'#064907'}}>Float</span> property: you can map only <span style={{color:'#064907'}}>Float</span> and <span style={{color:'#0039e3'}}>Integer</span>
- <span style={{color:'#7239e3'}}>Boolean</span> property: you can map only <span style={{color:'#7239e3'}}>Boolean</span> <span style={{color:'#ff9307'}}>variables</span>
- <span style={{color:'#ff9307'}}>DateTime</span> property: you can map only <span style={{color:'#ff9307'}}>DateTime</span> <span style={{color:'#ff9307'}}>variables</span> and hard-coded values with the format:&#x20;
  - `dd/MM/yyyy hh:mm:ss`
- <span style={{color:'#007364'}}>Data Model</span>: you can map only a <span style={{color:'#007364'}}>Data Model</span> variable of the same type. e.g. a “Customer” property accepts only the “Customer” data type as a variable. 
- <span style={{color:'#0671d6'}}>File</span>: you can map only another <span style={{color:'#0671d6'}}>File</span>.
