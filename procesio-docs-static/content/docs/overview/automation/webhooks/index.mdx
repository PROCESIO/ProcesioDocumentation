## What are webhooks?

Webhooks are user-defined HTTP callbacks. They are automated messages sent from external systems to PROCESIO when something happens in the external platform. Webhooks have messages (called payload) and are sent to a unique URL in PROCESIO.

Webhooks can be used to receive data from external apps and services when an event occurs. Also, webhooks can be used as triggers for process flows when we want to receive data and use it to kick off a process based on the data received. This allows several different services to connect to PROCESIO and run a process when data is received.

<Callout type="warn">
Please keep the webhook URLs private. They can be accessed directly by anyone having the link.
</Callout>

## Creating a webhook

Go to **Webhooks** page. By pressing **Create webhook**, the configuration side panel will open and you will see the following:

![](/images/docs/sUFBqPen9OJlhb6t7fYiu_image.png)

&#x20;1\. **Webhook Name** - this field is mandatory and it has a limit of max length 32 characters. 	&#x20;

<Callout type="info">
Webhook names must be unique.
</Callout>

2\. **Webhook URL** - generated by default when opening a new webhook configuration panel, this is the webhook URL that users must provide to the external platform when configuring the webhook there.

To configure your webhook, there are several options:&#x20;

- start listening and send an example data set to the unique URL from third-party services; the data model will be automatically created;
- manually add the payload to generate the data model.

![](/images/docs/5e4pO3q5qCM3LaSbjL0dD_image.png)

## Listening for data:

The **Configure Response** panel **Handshake response body** & Custom Response- offers the option to configure the response:

- Select the prefered configuration method from the dropdown:

![](/images/docs/2zH4qd0ayIoKJE8kqw4Uk_image.png)

- Configure the payload and copy the unique webhook URL and paste it in the webhook configuration in the external service/app:

![](/images/docs/vI72vxBPelWMcWrI4Uv24_image.png)

- Configure the respone in Handshake response body section:

![](/images/docs/kfVx2wrtka_0fudLLA_Zl_image.png)

- Listen the handshake by clicking the button. PROCESIO is now listening for the webhook data in order to determine the data structure automatically. Once you start listening, you will see this page until data is received or until the timer ends, if no data is received.



![](/images/docs/0rbeB-jtl2iKCn_nzXycn_3.png)

- Send the payload from the webhook configuration in the external service/app.
- Once the data is received, you will be able to see the tree structure of the created data model, collapse & expand attributes or search within the structure:

![](/images/docs/pW_V8W7g7PToh-EHQMWzm_image.png)

<Callout type="info">
“Listen for another event” will trigger the loading state again. You can generate in the meantime another event from the other platform. If the data structure of the event received is different, the attributes are merged and duplicates removed. The data structure is updated, growing incrementally.&#x20;
</Callout>

Configuration Panel - Custom process has a similar functionality with the Handshake, with the distiction that the Custom process configuration remains saved, while in the Handshake configuration resets after saving and exiting the process.

## Adding the payload manually

If you choose to manually define the webhook payload, you will see the following screen, where you can add your structure and generate the data model.&#x20;

![](/images/docs/ZgdOm1LUdsm3mxuYFKG69_5.png)

<Callout type="info">
The payload structure is taken from the external platform’s webhook documentation.
</Callout>

## Editing a webhook

You can edit an existing webhook by selecting the option from the contextual menu, but please be aware that you can only edit the webhook in the same way it was created.

![](/images/docs/uiixbElo-Giznn5JY0IsG_8.png)

If the webhook was originally created by listening for data, it can only be updated by listening for another event.

![](/images/docs/bms9sxT_Mad1Krb2FBiMM_7.png)

If the webhook was originally created by manually adding a payload, it can only be updated by manually manipulating the payload and regenerating the data model.

![](/images/docs/csUR1zdo_OV6r_3IQC-Ky_6.png)

## Deleting a webhook

You can delete an existing webhook by selecting the option from the contextual menu, but please be aware you can only delete a webhook that is not used in a process.

![](/images/docs/Imt1c3HjYk5AcM9Y3BAnJ_8.png)

<Callout type="info">
To delete a webhook, please remove it first from any processes that might be using it.
</Callout>

## Assigning a webhook

To trigger a process with webhooks, go to Process Designer and select the process to be triggered.&#x20;

From the top bar, open up the webhooks panel.

![](/images/docs/2K_8Gk-qZKMFWBzBbhjGM_screenshot-2022-01-05-at-094412.png)

![](/images/docs/qvrz4wIN-eGStpKvMnRyB_11.png)

You can select one webhook to kick off a process. Once you select a webhook from the dropdown list, it will populate the:

- local variable name
- webhook data model

**(Optional) Filter Rules&#xA0;**&#x61;llow you to filter or apply conditions on the data set that will be triggering the process.

![](/images/docs/vKl5MB9eqDkpQKmVYVMO0_13.png)

Here's an example. Let’s say you have an event “issue updated” that happens in Jira and the payload of a ticket is sent to PROCESIO. If there are no filter rules applied, all event payloads for “issue updated” will trigger your process, no matter what property has changed. 

But you may want only payloads with `name.old`*&#x20;!=&#x20;*`name.new` to be relevant, not all tickets that suffered an update within the Description. By using the complex condition builder, you can build a rule by creating conditions that use the attributes of the data model generated when saving the payload of the event. 

You can add unlimited rules and if the conditions are met, the process will be launched.

## Unassigning a webhook

To remove a webhook from a process, please open up the "Assign webhook" panel and, from the "Select webhook" field, remove the assigned webhook to clear the input.

![](/images/docs/mdRRgCDuSY3jJ4G99sMbr_14.png)
