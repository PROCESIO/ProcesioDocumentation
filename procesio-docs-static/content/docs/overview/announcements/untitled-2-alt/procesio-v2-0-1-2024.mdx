This release introduces new capabilities for workflow automation, improvements to platform usability, enhanced integrations, and several fixes aimed at increasing reliability and performance.

# ✨ New Features

## 📊 Table Control

A new **Table Control** component has been introduced to enable better data visualization within forms and applications.

The control supports:

- Pagination for large datasets&#x20;
- Sorting and filtering&#x20;
- Structured display of tabular data&#x20;
- Dynamic data binding within workflows&#x20;

This component enables users to display and manage structured data more effectively within PROCESIO interfaces.

## 🧩 NodeJS Scripting Action

A new **NodeJS scripting action** allows developers to implement custom logic directly within workflows.

This action enables:

- Advanced data transformations&#x20;
- Custom automation logic using JavaScript&#x20;
- Integration with external libraries and services&#x20;

## 📝 Forms & Tasks Designer Enhancements

The **Forms & Tasks Designer** has been enhanced with additional UI components and capabilities to improve form-building flexibility.

### 🧱 New Components

- **List Control** for displaying collections of items&#x20;
- **File Upload component** for attaching files during task execution&#x20;

### 🎨 Form Customization

- Support for **themes** to customize form appearance&#x20;
- Improved **approval step configuration** for workflow-driven tasks&#x20;

### ⚡ Form Events

New event support allows developers to trigger actions at different stages of form usage.

## 🔐 Credential Templates

New credential templates have been added to simplify integration with external services.

Supported templates include:

- GitHub&#x20;
- Azure Storage&#x20;
- Azure OAuth2&#x20;
- Microsoft integrations&#x20;

These templates standardize authentication configuration when connecting PROCESIO to third-party services.

# 🔧 Improvements

## ⚠️ Workflow Error Handling

Workflow error handling has been enhanced to provide greater control over process execution.

A new **Error Port** allows developers to define how workflows respond when errors occur.

This enables:

- Continuing process execution even when an action fails&#x20;
- Implementing custom error-handling logic&#x20;
- Improving workflow reliability and resilience&#x20;

## 🌐 API and Webhook Improvements

### 🔗 Call API Action

The **Call API** action has been improved with enhanced response handling and better support for integration scenarios involving external services.

### 📡 Webhooks

Webhook functionality has been enhanced to support improved response processing and more reliable integrations.

## 🧭 Toolbar Improvements

The platform toolbar has been updated to improve usability and navigation.

Changes include:

- Updated icons&#x20;
- Clearer action naming&#x20;
- Improved documentation for toolbar actions&#x20;
- Enhanced usability when managing folders and resources&#x20;

## 🐞 Error Diagnostics and Messaging

Improvements were made to the error reporting system to provide clearer diagnostics during process execution.

Enhancements include:

- Improved visibility of error messages in failed workflows&#x20;
- Better diagnostics during process execution&#x20;
- Clearer feedback when publishing processes with expired resources&#x20;

These improvements simplify troubleshooting and debugging.

# 🐛 Fixes

The following issues have been addressed in this release:

- Improved handling of **process execution metadata**&#x20;
- Fixes related to **process versioning**&#x20;
- Improved **execution date tracking**&#x20;
- Fixes for missing or incomplete **error messages during process failures**&#x20;

# 📬 Other Updates

## 📧 End Process Notifications

The **End Process notification email** now includes the **WorkspaceId**, providing better traceability when managing executions across multiple workspaces.
