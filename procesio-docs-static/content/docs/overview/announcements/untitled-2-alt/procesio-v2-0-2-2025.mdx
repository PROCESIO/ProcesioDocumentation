# 🚀 **PROCESIO 2025 Release Notes** 🎉

**A Year of Innovation, Performance & Reliability**

Get ready — 2025 was packed with upgrades across the platform: Forms, Process Execution, Credentials, Debugger, Global Architecture, Integrations, and more. Below is your consolidated release for the entire year.

# 🌟 **Front-End Enhancements**

### 🧩 Forms: Powerful New Capabilities

- Introduced **form applications** as entry points for workflows.&#x20;
- Added **global forms**, enabling cross-workspace reuse at scale.&#x20;
- New advanced controls: improved dropdowns, dynamic rows, instance IDs, copy-to-clipboard functionality, and export capabilities.&#x20;
- Improved **UI for required inputs**, validation feedback, and overall form builder clarity.&#x20;
- Enhanced **privacy management**, including improved “private form” screens.&#x20;
- Introduced **custom URLs** for publishing & accessing forms more flexibly.&#x20;

### 📊 Usability & Experience

- Added **workspace ID to links**, improving navigation & troubleshooting.&#x20;
- Improved **variable mapping UI** and nested Data Model mappings.&#x20;
- Boosted performance: **bundle sizes reduced by more than 5×**.&#x20;
- Introduced **Vue 3**, setting the stage for significantly improved performance going forward&#x20;



### 🛠️ Debugger Improvements

- Debugger now fully supports updating file variables and complex component testing.&#x20;
- Improved stability when stepping through processes with file variables and parallel structures.&#x20;
- Support for *restoring processes from instances* added.&#x20;

# ⚙️ **Back-End, Execution Engine & Performance**

### 🚀 Process Execution Enhancements

- Major optimization to **overall execution time**.&#x20;
- Improved handling of nested Data Model attributes in actions.&#x20;
- Introduced **HIPAA Mode – data retention controls**, secure output retrieval, and data-deletion rules.&#x20;
- Added **process timeout controls**, execution safeguards, and maximum process limits for stability.&#x20;
- Improved **AOM/AIM TTL handling**, reducing execution delays.&#x20;
- Refactored orchestration engine for faster action-level performance.&#x20;
- Major reliability upgrades to AWS S3, Azure Blob, and nested-file behavior.&#x20;

### 📈 Stability, Monitoring & Error Handling

- Improved graceful shutdown, retry policies, caching, and consumer recreation logic.&#x20;
- Enhanced error messages across publishing, execution, and connectors.&#x20;
- Better handling of missing variables, null values, decisional edge cases, and comparator logic.&#x20;
- Fixed inconsistencies in **runtime DateTime formats**, improving API compatibility.&#x20;

# 🔐 **Credentials & Security**

### 🛡️ Stronger Authentication

- Implemented **2FA at workspace level**.&#x20;
- Updated authentication workflow, reducing login issues.&#x20;
- Improved credential template redirection handling for multi-environment deployments.&#x20;

### 🔑 Credential Enhancements

- Enabled credentials to work seamlessly across multiple execution requests.&#x20;
- Added support for **new predefined credential templates** (REST, OAuth2, HashMAC, Microsoft ecosystem, etc.).&#x20;
- Improved security through **modernized encryption algorithms** and credential synchronization.&#x20;

# 🌐 **API, Connectors & Integrations**

### 🔄 Call API Enhancements

- Improved certificate handling and output processing.&#x20;
- Support for complex Data Model → API mappings.&#x20;
- Fixes for Webhooks, especially when handling files or empty bodies.&#x20;

### 🧩 Connectors

- Major updates to t**he custom connector framework**, including new event types&#x20;
- Better import/export, more robust connector validation&#x20;
- Improvements to sFTP/FTP, including path handling and uniformity updates&#x20;
- Added SQL parameters, SQL command timeout, and improvements to execution behavior&#x20;

# 📂 **File Handling & Document Generation**

- Improved handling of nested file structures.&#x20;
- Fixed multiple issues around **mapping files**, list-of-files behavior, or null file values.&#x20;
- Improved PDF and DOCX conversion engine, including diacritics support.&#x20;
- Enhanced HTML → PDF reliability.&#x20;
- Added new supported file extensions and validation improvements.&#x20;

# 📨 **Mailbox, Webhooks & Notifications**

- Upgraded mailbox reading with OAuth integration.&#x20;
- Improved submission events, UI notifications, and form-trigger behavior.&#x20;
- Improved Webhook execution, including child-process data retrieval and empty-body handling.&#x20;

# 👥 **User & Workspace Management**

- Allowed editing of first name and last name directly.&#x20;
- Improved workspace switching, logout logic, and workspace ID propagation.&#x20;
- Enhanced co-owner permissions and invitation flows.&#x20;
- Automatic cache cleaning on subscription changes, fixing stale issues.&#x20;

# 🧮 **Actions: Improvements & New Capabilities**

### 🚀 New / Improved Actions

- Word Mail Merge&#x20;
- HTML to PDF (with improved diacritics)&#x20;
- DOCX → PDF and DOCX → HTML conversions&#x20;
- Import/export improvements across flows and forms&#x20;
- List Contains, Concat, Subprocess mapping, and ListAdd stability updates&#x20;
- JavaScript, Python, PDF actions updated for reliability&#x20;
- Read Range from csv & split workbook to csv action, allowing faster handling of large Excel files by leveraging the csv file format
- CURL: Native HTTP Execution
  &#x20;Execute fully customizable HTTP requests using CURL syntax directly within workflows, enabling flexible API integrations without predefined connectors.
- Ruby: Advanced Scripting Capabilities
  &#x20;Run custom Ruby code in workflows to handle complex logic, data processing, and integrations with full control and flexibility.

### 🧹 Fixed or Enhanced

- Foreach iterator fixes&#x20;
- Delay-action behavior&#x20;
- Decisional logic: null comparison, JSON evaluation, contains/not-contains, multi-iteration issues&#x20;
- Get Recent Instances & Get Instance Outputs enhanced&#x20;
- Normalize Diacritics improvements (string handling)&#x20;

# 🗄️ **Data Models & Storage**

- Introduced HIPAA-compliant file retention rules.&#x20;
