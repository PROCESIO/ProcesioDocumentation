Typeform is a Spanish online software as a service company that specializes in online form building and online surveys. Its main software creates dynamic forms based on user needs. Procesio can integrate with Typeform in order to retrieve user inputs that can be then used for marketing purposes or to move them to a third party platform.

We are assuming that you already signup on [Procesio](https://procesio.com/) and [Typeform](https://www.typeform.com/). If not then first make an account and activate them.

### **Typeform**

&#x20; 1\.    Login to your [Typefom](https://www.typeform.com/) account.

&#x20; 2\.    Create a form in your wokspace, publish it and add a few entries. Then entries that you wish to be transferred need to be set as mandatory from the **Create** tab where you need to select the field and from the right side panel select **Required**.

![](/images/docs/pell7FQqQhNt-oqV1-RCR_image.png)

<Callout type="error">
Only forms that have been published and have data can be accessed by APIs.
</Callout>

&#x20; 3\.    From the upper right corner navigate to the **Settings&#x20;**&#x73;ection.

![](/images/docs/CUns-VR_76pnen9_vcWUA_image.png)

&#x20; 4\.    Select **Personal tokens** and generate a new token. **Save** the token as it will only be displayed once.

### **Procesio**

&#x20;   &#x20;**&#x20;I. Credential Manager**

&#x20; 1\.    From you Procesio account navigate to the **Credential Manager** section and click the +**Add new button**.

&#x20; 2\.    Provide a **Name** for you credential and select **Rest API Configuration** from the credential type dropdown. and click **Next step**.

![](/images/docs/i15CqNS8g7Qvrzn2NAJpN_image.png)

&#x20; 3\.    Add the necessary configurations.

- **URL\***: The base URL for making a request - https\://api.typeform.com/.
- **Method\*** - GET.
- **Test endpoint**: /me (endpoint that provides information about your account).
- **Authentication method**: API key authentication.
- **Key**: Authorization.
- **Value**: Bearer \&lt;the token generated in the previous section>.

![](/images/docs/EW2ZlEYRto5cQghai1PBy_image.png)



&#x20; 4\.    Click **Test connection** and if you receive the **Successfully tested!** message **Save** the connection.



&#x20;   &#x20;**&#x20;II. Process Designer**

**&#x20;&#x20;**&#x31;.    In the **Process Designer** section click the **Create new** button.

&#x20; 2\.    Give the process a name and drag the **Call API Action** to the canvas.

![](/images/docs/MngSj2J95g62TtDEtbg7z_image.png)

&#x20; 3\.    Create the <span style={{color:'#2166ae'}}>&lt;%FormResponses%&gt;</span> variable of type Json.

![](/images/docs/J4smB_pWWIcAJstW3RizV_image.png)

&#x20; 4\.    Click the Call API Action to access its configuration.

&#x20; 5\.    Select the api configuration we created on the first part of the Procesio section and click **Configure request**.

- **Verb\*** - GET.
- **Endpoint** - forms/\&lt;the form id>/responses.
- **Body output** - we will save the response in the variable we created in step 3.

![](/images/docs/UsXLJ24C2DaD4Y-uLMzOy_image.png)

&#x20; 6\.     **Save**, **Validate** and **Run** the process.

&#x20; 7\.     Click **Check instance** to view the result.

&#x20; 8\.     As you can see the form data was saved in the <span style={{color:'#2166ae'}}>&lt;%FormResponses%&gt;</span>.

![](/images/docs/ku_3SKl-H2K29ASNLvsuR_image.png)
