You need first to go to the [Azure Portal.](https://www.portal.azure.com) From there, you will need to go through the following steps:

### Select or Create a storage account

1. In the search bar, search for the Storage accountCreate or select a Storage account:

![](/images/docs/IkcAy5PMKAgqY_qmqIchI_image.png)

![](/images/docs/Og7R0zIY8SKqP-bb1FV6-_image.png)

&#x20;    2\. Create a new app dedicated to you PROCESIO usage:

Search for the App Registration in the search bar and select or create a new app.

![](/images/docs/768bMdsdhxVRHimNu6ile_image.png)

&#x20;    3\. After the app is created, select the app and from the left side meniu, select API permissions and in the Azure Storage Permissions select user impersonation and add the permissison:

![](/images/docs/tWgFuMNzLOc3miEQoUzWQ_image.png)

![](/images/docs/jAIzH-E8KI-EBbrePwsjQ_image.png)

After you have the user impersonation permission also select the Grant admin consent for Default Directory:

![](/images/docs/4dPQ4VCrbOg0Z0TyXvam0_image.png)

&#x20;    3\. Navigate to PROCESIO and create a OAuth2 credential:

![](/images/docs/It7vunMairvB3oIFil4-P_image.png)

- For the URL enter: https\://\&lt;AppName>.blob.core.windows.net/ ( Replace \&lt;AppName> with the name of the app created at step 2.
- For the Scopes add : **https\://storage.azure.com/.default offline\_access**
- In the Azure Portan navigate to the app overview:

![](/images/docs/VYjzBO36AxKoIk01Sa-fu_image.png)

- Copy the Application (client) ID and paste it in the PROCESIO credential Client ID field.
- For the client secretet, select the app created in Azure, on the left side meniu, navigate to **Certificates & Secrets** and create a new Client secret, then add it in the Client Secret field of the PROCESIO credential.
- For the URLs, navigate to the app overview in Azure Portal and click on the Endpoints, from there the first 2 endpoints are required in the PROCESIO credential ( authorization and token v2)

![](/images/docs/n0nahgwexTukdSB8q-mhb_image.png)

- Copy the OAuth Redirect URL from the PROCESIO credential and in the Azure Portal, select your app, navigate to Authentication, click on add a platform, select "web" and paste the OAuth URL.&#x20;

![](/images/docs/qMmyPuNLtSgtlpKr1d8c5_image.png)

![](/images/docs/UVyr5KZ7QH-pmDst9d9Ed_image.png)

&#x20;     4\. Click Save and authorize, login with your azure account.

### List blobs

The call API to list all files in the azure storage account:

url: ?comp=list&#x20;

no query params

headers: x-ms-version 2023-11-03

![](/images/docs/e3irzzvUEDGNVuO3s8dsp_image.png)

### **D**ownload file

The call API to download a file from the azure storage account:

The download file call api is similar to the list files, the only difference is the endpoint, where you have to specifiy the contaner and the name of the file you want to download. In the example below, the container name is fileupload, and the file is sample.pdf:
