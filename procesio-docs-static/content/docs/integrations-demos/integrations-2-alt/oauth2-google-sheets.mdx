The Google Sheets credential relies on the OAuth2 protocol. OAuth2 is an industry-standard protocol for authorization. It allows a client to easily authenticate and authorize access to resources.

##  How to get Google OAuth2 credentials

You will first need to go to the [Google Cloud console](https://console.cloud.google.com/). From there, you will need to go through the following steps:

### **Select or Create a project in Google Cloud Console**

1. If you don’t have a Google project yet, or if you need to create a new project dedicated to your PROCESIO usage, you will need to create such a project and fill in the required information.

![](/images/docs/aL-XxFbtMXiOfw0TBSmge_image.png)

### **Enable the Google Sheets API in your project**

<Callout type="info">
Note that if the Google Sheets API is already enabled in your project, it will appear listed in your dashboard.
</Callout>

&#x20;    2\. If you have not enabled the Google Sheets API in your project yet, click on the button “Enable APIs and services” > Search for the Google Sheets API and select it.

![](/images/docs/F6kB4_HA_2q5BUoE4LgwR_image.png)

&#x20;    3\. Click **Enable.**

![](/images/docs/WCRXTELkk1Ujjzm9uC9mz_screenshot-2023-06-28-at-141358.png)

### **Create credentials**

&#x20;    4\. Click “**Create Credentials**” to create a new credential for your Google Sheets API.

![](/images/docs/yMo6-gJWEudoeEASfpMws_image.png)

&#x20;    5\. Select  “**User data**” and click **Next**.

![](/images/docs/tEGXQyreb6YZRoymOY_92_image.png)

&#x20;    6\. Add a **name**, an **email** for support and developer contact information then save and continue.

![](/images/docs/OvtBJWX3TkZtCoBc3g8ni_image.png)

&#x20;    7\. In the **Scopes** section click **Add or remove scopes** and select the **Google Sheets API** scopes you need, then click **Update**.

![](/images/docs/grvF6C9Ohzo6QXhRLgucv_image.png)

&#x20;    8\. After you added the **Scopes&#x20;**&#x63;lick **Save and Continue**.

&#x20;    9\. In the **OAuth Client ID&#x20;**&#x73;elect **Web application&#x20;**&#x61;nd add a name for the application. After selecting Web application as application type, the section will expand displaying Authorized redirect URIs.

&#x20;    10\. Go in PROCESIO and Create an **OAuth 2** **Credential**.

&#x20;    11\. Copy the **OAuth Redirect URL&#x20;**(Automatically generated by PROCESIO when creating the Credential).

![](/images/docs/dQ9nRa4xedyOfTsYQbS2n_image.png)

&#x20;    11\. Return to **Google Cloud console&#x20;**&#x69;n the **OAuth Client ID&#x20;**&#x73;ection, and under **Authorized redirect URIs&#x20;**&#x63;lick **+Add URI** and paste the link copied from PROCESIO.

![](/images/docs/7JQoKphss4jhPP36eq-aU_image.png)

&#x20;    12\. Click **Create&#x20;**&#x61;nd then **Done**.

&#x20;    13\. Click on the **OAuth consent screen** in the right side panel, and under **Test users**, click **+Add Users** and enter your email address.&#x20;

<Callout type="info">
If the **Application&#x20;**&#x69;s created using an **enterprise email** within a specific organization:

- If the Application **User Type** is set to **Internal**, all users from that organization will have access to the App, and the Test users section will not be displayed.
- If the Application **User Type** is set to **External**, you should add the emails that need access to the App under **Test users**.
</Callout>

![](/images/docs/J0A5jg8qQjrQ1o-ocY9wO_image.png)

&#x20;    13\. Click on the **Credentials&#x20;**&#x73;ection in the right side panel, in the **OAuth 2.0 Client IDs** section, click on the newly created application.

&#x20;    14\. Copy the **Client ID** and **Client Secret**.

![](/images/docs/nNekJyowsqXPOQvM7Hzq-_image.png)

&#x20;    15\. Return to the **Create Credential** section in **PROCESIO&#x20;**&#x61;nd finish creating the credential. All the URLs needed can be found in the **API Documentation** of the app (Google Sheets in this case):

- &#x20;In the **URL** field you need to input the endpoint where the requests will be made:
  `https://sheets.googleapis.com/v4
   `&#x20;

- &#x20;In the **Scopes** field input the scopes you need to use:
  `https://www.googleapis.com/auth/spreadsheets.readonly`
  `https://www.googleapis.com/auth/spreadsheets`

- In the **Client ID** and **Client Secret** fields insert the values from the Credential created in Google Cloud. (copied at step 14)

- &#x20;In the **Auth URL** field you need to input the Authorization URL:
  `https://accounts.google.com/o/oauth2/v2/auth ` &#x20;

- &#x20;In the **Token URL** field you need to input the Token URL:
  `https://oauth2.googleapis.com/token`

![](/images/docs/Su7FbZjZPRrog1ZyITM2W_image.png)

![](/images/docs/0aNtFqIgFP0ybwvKp-R0-_image.png)

&#x20;    16\. Click "**Save and Authorize**" to generate your token.&#x20;

&#x20;    17\.  If the Google app created in the Google Cloud Console is in testing (no need to publish the app), you will receive the following prompt:&#x20;

<Callout type="info">
If the **Application&#x20;**&#x69;s created using an **enterprise email** within a specific organization and with the **User type** set as **Internal**, this prompt will not be displayed.
</Callout>

![](/images/docs/riqw1dGUzKVFFkfcoMHh1_image.png)

- Click **Continue**.

&#x20;    18\. In the following prompt check **Select all** and click **Continue**.

![](/images/docs/RbcGVE_qWZ4whPYNSOTes_image.png)

&#x20;    19\. In the **Test connection** section add any **GET&#x20;**&#x72;equest in order to confirm that everything was configured properly.

Endpoint: `/spreadsheets/{SheetId}/values/A1:B9`

- *Replace \&#123;SheetId&#125; with your Sheet Id&#x20;*

![](/images/docs/VtnrB117OO-1hbRK8w4r3_image.png)

&#x20;    20\. Click **Save** to save the credential.
