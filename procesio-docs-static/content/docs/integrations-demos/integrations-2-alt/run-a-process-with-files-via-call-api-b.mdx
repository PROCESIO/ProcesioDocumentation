How to run a process via Call API:

&#x20;    1\. Create a process in PROCESIO - Process Designer.

For this demo I created a process - Demo, with 2 input variables: a file and a string.

![](/images/docs/DjQKHfnjeWyFx1Y0Pu_JM_image.png)

&#x20;    2\. Create a API key in Automation - API Key.

![](/images/docs/EGWBFrOHo6qqmAzRteCpk_image.png)

### &#x20;    3. Send a Post Request to the endpont:&#x20;

`https://webapi.procesio.app/api/projects/{projectID}/instances/publish`

![](/images/docs/gtio1Mr0x0DSOumYxC-3h_image.png)

> curl --location 'https\://webapi.procesio.app/api/projects/projectIdValue/instances/publish' \\
> \--header 'workspaceId: WorkspaceIDValue' \\
> \--header 'key: ApiKeyName' \\
> \--header 'value: ApiKeyValue' \\
> \--header 'Content-Type: application/json' \\
> \--data '\[
> &#x20;   \&#123;
> &#x20;       "id": "fileIdValue",
> &#x20;       "value": \&#123;
> &#x20;           "name": "filenNameValue"
> &#x20;       &#125;
> &#x20;   &#125;,
> &#x20;       \&#123;
> &#x20;           "id": "VariableIdValue",
> &#x20;           "value": "VariableValue"
> &#x20;       &#125;
> ]'

<Callout type="info">
Follow the instructions below to replace the highlighted text.
</Callout>

Headers:

| Key         | Value               |
| ----------- | ------------------- |
| workspaceId | \&#123;worspaceID value&#125; |
| key         | \&#123;API key name&#125;     |
| value       | \&#123;API key value&#125;    |

- The project ID is displayed in the Process Designer under the name of the process.

![](/images/docs/DjQKHfnjeWyFx1Y0Pu_JM_image.png)

- The workspace ID is displayed in Dashboard:

![](/images/docs/rME8W9MpI7RaOyyE2Pdgn_image.png)

Body:

> \[
> &#x20;   \&#123;
> &#x20;       "id": "bf72a6c3-c5f1-4f0d-a071-88b6c93bfc3b",
> &#x20;       "value": \&#123;
> &#x20;           "name": "test.csv"
> &#x20;       &#125;
> &#x20;   &#125;,
> &#x20;       \&#123;
> &#x20;           "id": "ad3f088c-54c5-4537-96c2-3ad56c77c3c0",
> &#x20;           "value": "Test"
> &#x20;       &#125;
> ]

- In the body, the input variables must be initialized by ID with their values.&#x20;
- Navigate to Process Designer, open developer tools and go to Network, then Run the process.
- In the Publish request you can find the variables IDs for your process. ( In my case I have a file and a string ):

![](/images/docs/r7oSnVRs4YBwLKKVAvv2o_image.png)

![](/images/docs/Uy559VmKN-IqBbmBNVYlQ_image.png)

### &#x20;    4. Send a Post Request to the endpont:&#x20;

`https://webapi.procesio.app/api/file/upload/flow`

![](/images/docs/943109qgl4AomPuJUmKD1_image.png)

> curl --location 'https\://webapi.procesio.app/api/file/upload/flow' \\
> \--header 'Fileid: FileIdValue' \\
> \--header 'ProcessId: ProcessIdValue' \\
> \--header 'FlowInstanceid: InstanceIdValue' \\
> \--header 'Workspaceid: WorkspaceIdValue' \\
> \--header 'key: ApiKeyName' \\
> \--header 'value: ApiKeyValue' \\
> \--header 'Variablename: VariableNameValue' \\
> \--form 'package=@"/FilePath"'

<Callout type="info">
Follow the instructions below to replace the highlighted text.
</Callout>

Headers:

| Key            | Value                     |
| -------------- | ------------------------- |
| workspaceId    | \&#123;worspaceID value&#125;       |
| key            | \&#123;API key name&#125;           |
| value          | \&#123;API key value&#125;          |
| fileId         | \&#123;file Id value&#125;          |
| projectID      | \&#123;project Id value&#125;       |
| FlowInstanceID | \&#123;Flow Instance Id value&#125; |
| Variablename   | \&#123;Variable name value&#125;    |

- File ID and Flow Instance ID are displayed in the response from the Publish api Call:

File ID: `$.variables[?(@.name == 'filename')].defaultValue.id `

![](/images/docs/rm9sq_rjENk7CS1vYUphf_image.png)

Flow InstanceID: `$.id `

![](/images/docs/rphvqPYxQ7D2OvRcaL6FC_image.png)

Body:

| key     | Value   |   |
| ------- | ------- | - |
| package | \&#123;file&#125; |   |



![](/images/docs/08TWuoPpS_eUKcCajnoJk_image.png)

### &#x20;    4. Send a Post Request to the endpont:&#x20;

`https://webapi.procesio.app/api/Projects/instances/instanceID`



![](/images/docs/6Pw1dKV7UrKFKNhYXqbBD_image.png)

> curl --location 'https\://webapi.procesio.app/api/Projects/instances/FlowInstanceID/launch?runSynchronous=true\&secondsTimeOut=300' \\
> \--header 'key: ApiKeyName' \\
> \--header 'value: ApiKeyValue' \\
> \--header 'workspaceId: WorkspaceIdValue' \\
> \--header 'Content-Type: application/json' \\
> \--data '\&#123;
> &#x20;   "ProcessID": "ProcessIdValue"
> &#125;'

Params:

| Key            | Value |
| -------------- | ----- |
| runSynchronous | true  |
| secondsTimeOut | 300   |

Headers:

| Key         | Value               |
| ----------- | ------------------- |
| workspaceId | \&#123;worspaceID value&#125; |
| key         | \&#123;API key name&#125;     |
| value       | \&#123;API key value&#125;    |

&#x20;Body:

> \&#123;
> &#x20;   "ProcessID": "\&#123;ProcessIdValue"
> &#125;

![](/images/docs/5hlc0FBt-mVIXeuLX-jjN_image.png)

&#x20;    5\. Check Project Instances and verify if the process was completed:

![](/images/docs/Y_9r2f92BcWY2mIHWEsi9_image.png)
