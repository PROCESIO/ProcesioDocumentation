To automate repetitive departmental tasks in your organization, Procesio can be the glue that holds everything together bringing together functionalities from different platforms to automate your workload and saving your team time.

In this example we will go through the implementation of a tool that could complement your hiring process with the aid of Webflow and Airtable. We will assume that you already have [Airtable](https://airtable.com/) and [Webflow](https://webflow.com/) accounts.

We will use Webflow for the creation of a form on your website that will send data regarding the results of the interviewing process and the interviewee contacts into a table in Airtable.

### **Airtable**

&#x20; **Step 1.**     We will create the table meant to receive the data.

![](/images/docs/2Qt3izIOQI07xBXbmOVyA_image.png)

&#x20;   **Step 2.**    Generate the Api key from your [Airtable account](https://airtable.com/account)**.**

![](/images/docs/NRWYzPIW4TwWQXlr-efvY_image.png)

### **Procesio**

&#x20; **Step 1.**    From the **Credential manager** create the connection intended to be used for Airtable.

![](/images/docs/rpcG5BETUrkJVCGg1FuA9_image.png)

<Callout type="info">
Replace the Test endpoint with a Get endpoint from your project found [here](https://airtable.com/).
Replace api\_key with your api\_key, that you can find [here](https://airtable.com/account).
</Callout>

&#x20;   **Step 2.**    [Create the process](/docs/integrations-demos/integrations-2-alt/integrations/demo-creating-your-first-process-flow) that will add data to Airtable.

![](/images/docs/_zmS3HoJMw5Sei3wDhAaJ_image.png)

&#x20; **Step 3.**    In this process we will use the following [variables](/docs/overview/processes/process-designer/working-with-process-variables): 

- Mandatory input variables:
  - Name
  - Email
  - Phone
  - Notes
- Optional output variables:
  - status
  - body

![](/images/docs/ABdTefhreh-NpmKgw03B6_image.png)

&#x20; **Step 4.&#x20;**&#x20;  For this process we will use the connection we just created with the **Post** verb and the [Create Applicants records API](https://airtable.com/api) provided by Airtable and add the necessary **Query Params** and body data. You will use the variables in the **body section** to send the data dynamically.

![](/images/docs/0uaCDDC4mHietIXO4mOPh_image.png)

```json
      {
              "records": [
                 {
                    "fields": {
                    "Name": "<%Name%>",
                    "Age": "<%Age%>",
                    "Email": "<%Email%>",
                    "Phone": "<%Phone%>",
                    "Notes": "<%Notes%>"
                } }
                      ], "typecast": true
           }
```



**&#x20; Step 5.**&#x20;**&#x20;  Save** and **Validate** the project.

&#x20; **Step 6.**    **Save the ID&#x20;**&#x6F;f your project for use it in the code you will add to Webflow. You can find it in the Process designer under its name or in the last part of the URL while inside the project

![](/images/docs/xzOjAWmhggHjJOcxDCLrK_image.png)

![](/images/docs/mtWlY2Ch8g1i6FsSVR_zD_image.png)

### **Webflow**

&#x20; **Step 1.**    Create your new application/ website from the **Webflow dashboard**

&#x20; **Step 2.**    Add a form on one of the pages.

![](/images/docs/Ue4qHlPt2EabzlxVIlDZn_image.png)

&#x20; **Step 3.**    From the left side of your Dashboard selec&#x74;**&#xA0;Pages**, and click the widget of the page containing your form to **Edit page settings.**

&#x20; **Step 4.**     In the **Before body tag** section you have the option to enter your code to manage the process you just created in Procesio.

- Here you will store the data entered in the form in variables that will be forwarded to Procesio
- Authenticate to Procesio
- Use the API calls provided here, in this case we will use the **/launch** endpoint.

The snippet of code used in Webflow can be found [here](https://github.com/PROCESIO/PROCESIO-Code-Examples/blob/main/Use%20cases/Webflow%20Procesio%20Airtable).
