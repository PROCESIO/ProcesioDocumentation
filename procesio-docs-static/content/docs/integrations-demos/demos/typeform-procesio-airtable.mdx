In this article I'll show you step-by-step how to automatically send Typeform responses to a base in Airtable using Procesio.

To begin, create and sign in to your accounts on [Procesio](https://procesio.com/), [Airtable](https://airtable.com/) and [Typeform](https://www.typeform.com/).

## ****[Typeform](/docs/integrations-demos/integrations-2-alt/integrations/typeform)****

&#x20; 1\.    In your Typeform workspace create a form. I used the Candidate Experience Survey template with the following entries:

![](/images/docs/lS-JAFwC-cNKdy75inZiA_image.png)



&#x20; 2\.    **Publish** the form and add at least one entry to test.

![](/images/docs/CbRNfew3Xk5ar0pxUc4Nh_image.png)

<Callout type="error">
Only forms that have been published and have data can be accessed by APIs.
</Callout>

Mark the fields that you want to export as mandatory. In the **Create&#x20;**&#x74;ab click the field and from the left side panel check **Required**.

![](/images/docs/sNTmQdtlKgC-pDz0-vo3J_image.png)

&#x20; 3\.    From the upper right corner navigate to **Settings**.

![](/images/docs/7b9vLfr_AQnk8HYtvQ-tB_image.png)

&#x20; 4\.    Select **Personal tokens** and generate a new token. **Save** the token as it will only be displayed once.

## ****[Airtable](/docs/integrations-demos/integrations-2-alt/integrations/airtable)****

## **Procesio**

### &#x20; **I. Create the credentials**

In your Procesio account navigate to **Credentials Manager** and click **Create new**.

**&#x20; 1.**&#x20;**&#x20;  Typefom**

a. Name the credential, select **Rest API Configuration** from the credential type dropdown. and click **Next step**.

![](/images/docs/D5D9JNpv6SOXSvnYd2ovz_image.png)

&#x20; b. Configurations:

- **URL\***: The base URL for making a request - https\://api.typeform.com/.
-
  **Method\*** - GET
-
  **Test endpoint**: /me (endpoint that provides information about your account)
-
  **Authentication method**: API key authentication
-
  **Key**: Authorization
-
  **Value**: Bearer \&lt;the token generated in the Typeform section>

![](/images/docs/Pkocnv-PqN_I-oc_reu4Q_image.png)

&#x20; c. Click **Test connection** and when you receive the **Successfully tested!&#x20;**&#x6D;essage, **Save&#x20;**&#x74;he connection.

&#x20; **2.&#x20;**&#x20;  **Airtable**

&#x20; a. Provide a Name for you credential and select **Rest API Configuration** from the credential type dropdown. and click **Next step**.

![](/images/docs/mkje5F9EqALbe2Ndk7Fiu_image.png)

&#x20;   b. Add the necessary configurations.

- **URL\***: The base URL for making a request - https\://api.typeform.com/.
- **Method\*** - GET.
- **Test endpoint:** Replace the Test endpoint with a Get endpoint from your project found [here](https://airtable.com/).
- **Authentication method**: API key authentication.
- **Key**: Authorization.
- **Value**: Bearer \&lt;the token generated in the Airtable section>.

![](/images/docs/Y0Y_BrQhxYFiHPO1nhHav_image.png)

&#x20; c. Click **Test connection** and if you receive the **Successfully tested!** message **Save&#x20;**&#x74;he connection.

### **III. Create the webhook**

&#x20; 1\.    In Procesio go to **Navigation**, select **Webhooks**, and click **Create Webhook**.

![](/images/docs/HXNrR50kpOHMR2NCNcqf8_image.png)

&#x20; 2\.    Give your webhook a name, copy the **Webhook url\*** and click **Start listening**.

&#x20; 3\.    In your Typeform account select the form and navigate to **Connect -> WEBHOOKS**.

![](/images/docs/MPA3F6lKnmuX1s5r8jv6a_image.png)

&#x20; 4\.    Click **Add a webhook** and add the link you copied from Procesio. Click **Save webhook**.

![](/images/docs/4rjbLGrPT97gnuoG5N_0t_image.png)

&#x20;  5\.    Make sure the webhook is set to **ON**.

![](/images/docs/wkW97Vmk9zXRssw-NbHAO_image.png)

&#x20;  6\.    Add a new test entry in your form and check that Procesio received the confirmation.

![](/images/docs/a9MyxgSj2cfGD4OKDlEgV_image.png)

&#x20;  7\.    Click **Save**.

### **II. Create the process**

&#x20; 1\.    Navigate to the **Process Designer** and click **Create new**.

&#x20; 2\.    Name the process, drag a **Call API Action** (to connect to Airtable) and a **JSON Mapper** action for each data field that you wish to retrieve from Typeform.

![](/images/docs/zXzwiP0IIFRWr19ZZg6iI_image.png)

&#x20; 3\.    Click the webhooks widget in the upper right corner, select the webhook that we created in the previous section, and add the condition to determine whether the process will run or not.

![](/images/docs/0f3LqwqWBpeUYqagu2Lya_image.png)

&#x20; 4\.    Create the process [process variables](/docs/overview/processes/process-designer/working-with-process-variables):

- ****<span style={{color:'#2166ae'}}>&lt;%Typeform%&gt;</span> - the input variable generated when we add the webhook to store the Typeform data.
- &#x20;&lt;%department%&gt;, <span style={{color:'#2166ae'}}>&lt;%name%&gt;</span>, <span style={{color:'#2166ae'}}>&lt;%punctual%&gt;</span>, <span style={{color:'#2166ae'}}>&lt;%email%&gt;</span>, <span style={{color:'#2166ae'}}>&lt;%rating%&gt;</span>**&#x20;-&#x20;**&#x74;he variables that will store the form inputs in Procesio.
- ****<span style={{color:'#2166ae'}}>&lt;%status%&gt;</span>, <span style={{color:'#2166ae'}}>&lt;%body%&gt;</span> - the helper variables that will help us diagnose the Airtable API Call in case of an error.&#x20;

![](/images/docs/wuX6w-nyJ4CpD-HSrZfJE_image.png)

&#x20; 5\.    Configure the **JSON Mapper** actions.

- give the action a name that will make it easier to recognize.
- click the action to select it and access its configurations.
- add the <span style={{color:'#2166ae'}}>&lt;%Typeform%&gt;</span> variable as the **Input Json\***.
- In **Query\*** add the corresponding JQuery expression (keep in mind that the data you search in is stored in the \&lt;%Typeform%> variable):
  - Get Department -> $..answers\[0].text
  - &#x20;Punctuality -> $..answers\[1].choice.label
  - Get Name -> $..answers\[2].text
  - Get Email -> $..answers\[3].email
  - Interview Rating -> $..answers\[4].number&#x20;

![](/images/docs/BNyKuZuebjgO8uJvbjTbz_image.png)

&#x20; 6\.    Configure the Airtable action.

- select the Airtable connection you created  and click **Configure Request**.
- select the **Post&#x20;**&#x76;erb.
- enter the endpoint for your table (you can find it [here](https://airtable.com/api)), I used /app1tmHLNGYKcrQxE/InterviewerFeedback.
- enter the query parameter **Content-type: application/json**.
- for our example we used the following **Body**.

```json
{
              "records": [
                 {
                    "fields": {
                    "Department": "<%department%>",
                    "Punctuality": "<%punctual%>",
                    "Name": "<%name%>",
                    "Email": "<%email%>",
                    "Interviewer(s) rating": "<%rating%>"
                } }
                      ], "typecast": true
}
```

![](/images/docs/pR-_HhsJ38kb1nBP-pZkg_image.png)
